记录日常使用到的 VSCode 插件

<!-- more -->

!!! note

    插件不在于多，而在于满足自己需求。要能分清是想要，还是需要。

### 在用插件

| 插件名称                | 作用                                                                 |
|------------------------|----------------------------------------------------------------------|
| filesize               | 在状态栏显示当前文件的大小                                           |
| GitHub Copilot Chat    | 提供基于 AI 的聊天式编程辅助，支持代码生成与问题解答                 |
| Markdown Auto Space    | 自动在中文与英文、数字之间插入空格，优化 Markdown 排版                |
| markdownlint           | 对 Markdown 文件进行语法和风格检查，确保符合规范                     |
| Pylance                | 提供高性能的 Python 智能感知、类型检查和代码导航功能                 |
| Python                 | 官方 Python 扩展，提供基础语言支持（如运行、调试、环境管理等）       |
| Python Debugger        | 基于 debugpy 的调试器，支持断点、变量监视等 Python 调试功能          |
| Ruff                   | 快速的 Python 代码 lint 与格式化工具，替代传统 linter 并提升性能     |
| Material Icon Theme    | 为 VSCode 提供基于 Material Design 的图标主题，提升文件和文件夹的视觉识别度 |
| Pride Themes           | 基于 LGBTQ+ 骄傲旗帜设计的 VSCode 主题，支持多种颜色风格的界面配色 |

### 曾用插件

| 插件名称 | 作用 | 不用理由|
|------|------|------|
|vscode-pdf| VS Code 中预览 PDF 文件| 不太在 VSCode 查看 PDF 了|
|Even Better TOML| 增强对 TOML（Tom's Obvious, Minimal Language） 文件的编辑体验|编辑 TOML 配置文件情况较少|
