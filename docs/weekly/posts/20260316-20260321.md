## 引言

这是我改革自己信息流后的第一篇周刊。目的是将自己一周内所见所想总结一下，带有些许个人色彩。

借助这个周报，顺道锻炼自己的信息提取和有效表达的能力。目前阶段只是寻找自己想要的东西，仅此而已。

## 🦞 OpenClaw

这一周颇为热闹，由 OpenClaw 引发的一系列多米诺骨牌效应开始显现。或者说一直以来大家始终没有找到一个合适的介质来承载 AI 大模型的能力。

OpenClaw 的出现，让个人 Agent 助理的实现再一次有了希望。

我还发现一个有趣的现象：好多工具都开始支持 CLI 了，都是受到 OpenClaw 的影响。从大家刚开始使用 Cursor IDE，到现在都在使用 Claude Code 和 CodeX 等终端工具。这背后反映了一个变迁：大家更信任 Agent 编程工具了。

腾讯、阿里和字节在一周内迅速建立了自己的“OpenClaw”。虽有些重合，却又各自不同。

腾讯的 QClaw 侧重 C 端服务能力的封装，毕竟腾讯手握微信，具有天然的优势。小龙虾的出现，让其有了抓手。用户期待微信 AI，本质是对“超级入口+AI 能力”的想象空间。但是感觉腾讯这方面做的有些克制，一部分原因在于庞大的用户和生态，微小的错误会被放大很多。

阿里的悟空是电商交易流程的封装。阿里底色是电商，主场也是这里。

字节的 ArkClaw 更多的是侧重企业标准化场景操作能力的封装，以飞书为主场地。

## 短剧 AI 化

DataEye 数据显示，2026 年 1 月份漫剧百强榜中，AI 仿真人短剧占比已经从去年的 7% 提升至 38%。短剧 AI 化，已是大势所趋。

但是好的内容还是关键。短剧经过野蛮发展后，内容逐渐同化，观众也逐渐展现审美疲劳。后续就发展就要看好的内容来吸引观众了。

AI 短剧的优势在于可以降低成本，提高制作效率。但是缺乏真实情感。最重要的还是好的内容。

生产力的提升必然引起新的变革。如何在这个变革中活下去，就需要找到哪个不变量。我姑且认为那个不变量就是好的内容。

## AI 提高效率？

AI 的出现，到底有没有提高我们的开发效率呢？这是一个很有趣的问题。我觉得最主要的是怎么用 AI 编程模型了。

未来会出现两级分化：少数人把 AI 当作学习工具，用它来理解底层原理。这喜人会比以前更快地变得卓越。而大多数人会选择不去理解，因为“反正结果已经有了”。

反观自己，我正在往大多数人这里移动。我最赞同的一个观点是：AI 是成长的加速器，不是理解力的替代品。

现阶段大家都在互相制造焦虑，每天不消耗 Token 做点事情，感觉就落后了。事实是：大部分人都在说谎。

能否提高整体的开发效率，最重要的是：优化的是不是瓶颈。大多数情况是：你没有提速，而是制造了一场“交通堵塞”，然后管它叫“生产力提升”。

## 工具

### PixPin

简介: 截图 / 贴图 / 长截图 / 图片转文字 (OCR) | 免费全能截图工具。

官网：https://pixpin.cn/

### Raycast

简介：一套功能强大的生产力工具集，全部集成于可扩展的启动器中。

官网：https://www.raycast.com/

这周将 macOS 的启动工具从 Alfred 转到了 Raycast 上。从这一周使用来看，个人感觉 Raycast 更好用一些。和 Alfred 相比，稳定性上还是差些，不过也算值得期待吧！

### markdown-auto-space

markdown-auto-space：在保存或格式化时，按可配置规则整理 Markdown 中的空格（中英混排、中文文案排版指北空格相关约定等）。

URL: https://marketplace.visualstudio.com/items?itemName=SWHL.markdown-auto-space

这是自己开发的第一个 VS Code 插件，详情可以看 [我开发的第一个 VSCode 插件](../my_first_vs_plugin.md)。

### DissionPage

DissionPage: 一个基于 Python 的网页自动化工具。既能控制浏览器，也能收发数据包，还能把两者合而为一。可兼顾浏览器自动化的便利性和 requests 的高效率。功能强大，语法简洁优雅，代码量少，对新手友好。

URL：https://drissionpage.cn/

最大的优点在于不用像 Selenium 安装不同版本的驱动。DissionPage 自动适配不同版本的浏览器。
