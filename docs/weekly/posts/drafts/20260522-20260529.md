## 摘录

### 一些思考

前些时间，我还有看到有些企业或者机构在担心数据泄露的问题。现在一点也看不到了。大家争相鼓励员工使用 AI 模型，而公司内部不会审查哪些东西会被员工喂给外部 API 接入的 AI 模型。

这个时代下，隐私已经不再那么重要了。还有的就是各种语音输入法的兴起。大家似乎都在涌向效率的一端，全然不顾其他隐患。也许此刻正是做网络安全企业的机会。

---

最近也看到一些优秀的 AI 短视频作品，这些作品看起来已经没有了前期的稚嫩，不管是情节、画质和特效都不输现有的作品了。让我想到了我们借助可灵 AI 或者 Seedance 2.0 等工具拍摄像《爱，死亡和机器人》这种故事短片。

我有一段时间特别爱看科幻小说，王晋康、何夕和刘慈欣的看的最多。他们有很多短篇作品，非常适合用 AI 模型拍成中国版的《爱，死亡和机器人》。

---

学到一个新词：功不唐捐，意为所下的功夫、付出的努力不会白费，必有所回报。正好契合了我一直以来的观点：当下做的事情，不一定会在现在看到回报。还想到一句话：善良是一场有序的轮回。我们只管：但行好事，莫问前程就好了。

---

边享受边做事会让他们觉得心里不够踏实，但我认为强迫自己做不开心的事情是很难长久的。单靠毅力并不能从根本上解决这个问题。一件事如果一直是坚持做的状态，必然做不长久的。

你只有实际尝试去做一件事，才会清楚自己到底喜不喜欢。例如，你想坚持每天运动，那就要先去尝试，慢跑也好打网球也好，只有尝试了，你才能了解自己喜欢什么运动。这样一来，你的身体反应和心理感受会给你一些反馈，如“慢跑让我觉得特别开心”“打网球并没有我想象的那么有趣”。

很多时候，我们往往会忽略去感受做某件事的感受。

### [你如何证明自己用 vibe coding 比之前古法编程效率高？](https://daily.zhihu.com/story/9789978)

这篇文章作者从维护开源项目的变迁生动地说明了当前 AI 编码带来的影响。很多观点我深深表示赞同。下面我会节选一些：

> GitHub 上偶尔有人提 issue，但真正能提交高质量 pull request 的人极少，能直接合入主干的更是罕见。
>
> 这其实是学术开源软件中一个非常普遍的结构性困境。

我想说的，不仅仅是学术开源软件，其他的开源工具也是一样的困境。一个 PR 是怎么产生的呢？

我的角度来看是：因为某种需求，我找到了这个开源工具，试用了一下，发现有个小 bug：传入中文路径时，程序会崩溃，但是我自己本地通过修改源码已经解决了。

考虑到我后续打算集成该工具到当前项目中，总不能每次更新，我都要记得修改这个工具的源码吧！于是，我就给这个开源工具提了一个 PR，修复这个问题。后续我用的时候就可以直接安装就行了。

很多小伙伴都是这样的场景参与到开源项目的构建和维护中来的。

> 最近几年，它们几乎停止了更新。不是没人用了，恰恰相反，用户一直在，需求一直在；而是能维护它们的人太少了。现在它们终于又开始演化了，这让我非常兴奋。但紧接着，第二个念头就冒了出来，而且挥之不去：如果模型能做这些，那我以前带过的那些学生呢？过去这些软件是我和博士生、本科生一起开发打磨出来的，以后还有学生能通过参与这样的项目成长起来吗？
>
> 这篇文章想讲的，就是这两件事之间的张力。AI 让我的软件重新活了过来，但也让我开始怀疑，那条我自己走过的、也带着学生走过的路，是不是正在消失。

很有同感。企业界对初级工程师的培养的路也面临消失的风险。

> 如果你让它从零开始独立设计一个这样的系统，或者对一个复杂系统做大规模的架构重构，以目前的能力来说，它做不好。它最强的场景是：人已经定义好了方向和边界，它在这个框架内执行。

这是对 AI 编码的清晰认知。我也是这么用的。
