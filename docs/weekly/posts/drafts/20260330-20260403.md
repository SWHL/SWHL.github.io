## 引言

又到了一个周末的时间了。时间如果按照周来走的话，个人感觉还是挺快的。

## Claude Code 源码泄露

> 在发布 npm 包 `@anthropic-ai/claude-code v2.1.88` 时，误将一个约 60 MB 的 .js.map（Source Map）调试文件打包进生产版本。
>
> 该 Source Map 文件本应在构建后被排除，但由于 .npmignore 配置缺失，导致包含 1900 多个 TypeScript 源文件、总计 51.2 万行代码 的完整项目结构被公开暴露。

Claude Code 主要开发者鲍里斯切尔尼 (Boris Cherny) 的一句话让我很有感触：

> 与其他任何事件一样，解决问题的反直觉方法是寻找更快的方法，而不是引入更多流程，在这种情况下就是要提高自动化程度，并且由 Claude Code 检查结果。

我已经将我维护的开源项目单元测试和发包步骤全部由 Github Actions 来实现了。重复的工作尽量减少手动操作。因为人类很容易在重复事情中犯错。人类也是流程中最容易带来不确定性的变量。
