### [这边清华劝退“数学天才”，那边北大校友刚创造历史](https://www.huxiu.com/article/4875747.html?f=rss)

「考试的时候遇到问题，因为时间有限制，就会把课本上或者老师教过的方法给试一遍，但自己做研究的时候，因为没有时间限制，我可以花很长时间去想一个问题，把自己脑袋里乱七八糟的想法都试一遍」。

### [写了 17 年开源代码，我为什么认为 Coding Agents 堆功能是在瞎折腾？](https://www.infoq.cn/article/sLVv23TfVFxoBPjNwcZI)

现在大多数 Coding Agent harness 基本分两种模式：要么 agent 想干嘛就干嘛，要么每一步都要问你：“你确定要删这个文件吗？”“你确定要列出这个目录吗？”……看似安全，但现实是，这种机制只会带来疲劳。用户要么直接关掉这些确认，开启 YOLO 模式，要么就无脑按回车，根本不会看提示。所以这并不是一个真正有效的解决方案。

---

我自己在编写代码时，就是这种状态。一个劲按回车，最后确认功能是否可用。至于是否阅读代码，那肯定是不会的。
